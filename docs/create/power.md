# Create：回転速度・動力制御（Power / RPM）

このページでは、Create MODにおける**回転速度（RPM）と動力制御**について、
サーバー参加者（クライアント側）が理解しておくべき基本概念と注意点をまとめています。

Createの機械は、**速ければ良いわけではありません**。  
速度・負荷・構成のバランスが重要です。

## 回転速度（RPM）とは

RPM（Revolutions Per Minute）は、  
シャフトや歯車が**1分間に何回転するか**を表します。

- 多くの機械は最低RPMを要求する
- 上限RPMを超えると停止・破損することがある
- 高RPMほど負荷が増える

## トルクと負荷の考え方

Createでは、明示的に「トルク」という数値は表示されませんが、  
**接続されている装置の数と種類**によって負荷が決まります。

負荷が大きすぎると、以下の状態になります。

- 回転が極端に遅くなる
- 完全に停止する
- 動力源側が止まる

## ギア比による速度制御

歯車のサイズ差を使うことで、回転速度を制御できます。

### 小歯車 → 大歯車

- 回転速度：低下
- トルク：増加
- 安定性が高い

### 大歯車 → 小歯車

- 回転速度：上昇
- トルク：低下
- 過負荷・破損に注意

## 安全なRPM管理の目安

以下は一般的な目安です。

- 低速（～16 RPM）
  - 農業系装置
  - 安定稼働重視

- 中速（32～64 RPM）
  - 加工・搬送装置
  - バランス型

- 高速（128 RPM以上）
  - 高速生産ライン
  - サーバー負荷に注意

## 回転方向と制御

- 歯車を1枚噛ませるごとに回転方向が反転
- クラッチやギアシフトで制御可能
- 方向ミスは動作不良の原因になりやすい

## よくあるトラブル例

- 速度を上げすぎて全体が停止
- 一部装置だけ動かない
- 接続数増加による負荷オーバー

トラブル時は、**まずRPMを下げる**のが基本です。

## サーバー環境での注意点

- 極端な高速回転構成は制限対象になることがある
- 無限増速構成は禁止される場合あり
- 大規模ラインは分割推奨

Createの動力制御を理解すると、  
**壊れにくく、止まりにくい工場**を作れるようになります。
