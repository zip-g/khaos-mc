# Create：サーバー利用時の注意事項（Caution）

このページでは、本サーバー（Fabric 1.20.1）において  
**Create MODを使用する際に必ず守ってほしい注意点**をまとめています。

Createは非常に自由度が高いMODですが、  
構成次第では**サーバー負荷・ラグ・不具合の原因**になります。

参加者全員が快適に遊ぶため、必ず一読してください。

## 高速回転構成について

Createでは歯車比によって無制限にRPMを上げることが可能ですが、  
以下のような構成は**非推奨または制限対象**です。

- 常時128 RPM超で稼働する装置
- 意図的な無限増速ループ
- 見た目目的以外の極端な高速回転

特に以下はサーバー負荷が高くなります。

- 高速回転＋大量のベルト
- 高速回転＋大量の可動ブロック

基本方針として、  
**必要最小限のRPMで設計してください。**

## 可動構造物（ムービング構造）

以下の構成は負荷が高くなりやすいため注意が必要です。

- 大型の回転アーム
- 多数のブロックを含む移動構造
- 常時稼働する回転式ファーム

対策として、

- 構造はできるだけ小さく
- 常時稼働ではなくON/OFF制御
- 使用後は停止させる

を心がけてください。

## クラッチ・スイッチの使用を推奨

Createでは、**動いていない装置にも負荷がかかる**場合があります。

そのため、

- クラッチ
- レッドストーン制御
- ギアシフト

を使い、  
**必要なときだけ動かす設計**を強く推奨します。

## ベルト・搬送量について

- 長距離ベルトの多用
- 大量アイテムの高速搬送
- 常時詰まりかけの状態

これらはTPS低下の原因になります。

対策例：

- 搬送距離を短くする
- 中間処理を分割する
- 必要以上に速くしない

## ワールド常駐装置に関する注意

以下のような装置は、  
**プレイヤーが不在でも動き続ける**ため要注意です。

- チャンクローダー付きCreate装置
- 常時稼働の自動ファーム
- 無人状態での大量処理ライン

無人稼働を前提とする場合は、  
必ず規模を抑え、動作確認を行ってください。

## トラブルが起きた場合

以下の症状が出た場合は、Create構成が原因の可能性があります。

- 周囲だけ極端に重い
- ログイン時にラグが発生
- 機械が急に止まる

その場合は、

1. RPMを下げる  
2. 構成を簡素化する  
3. 一時的に停止する  

を行ってください。

## 管理側対応について

以下の場合、管理側で対応を行う可能性があります。

- 明確に負荷原因となっている構成
- 注意後も改善が見られない場合
- サーバー全体に影響が出ている場合

対応内容には、

- 装置の停止
- 一部撤去
- 構成変更の依頼

が含まれます。

Createは工夫次第で十分高効率になります。  
**サーバー全体の快適さを最優先**で楽しみましょう。
